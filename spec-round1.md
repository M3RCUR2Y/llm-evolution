# LLM Evolution Timeline — Round 1 改进方案

## 核心目标
把"静态信息图"升级为"交互式可视化产品"

## Round 1: 修复最痛的问题

### 1. 节点防重叠
同月多模型时，垂直错开排列（像泳池里的人各占一条泳道）。
- 同一公司行内，同月模型上下交错
- 间距至少 28px

### 2. 搜索框
顶部加一个搜索框，实时过滤：
- 输入时高亮匹配的节点，其他变暗
- 支持中英文模糊搜索
- Esc 清除搜索

### 3. 提升信息密度
- 时间轴左侧加一条"累积模型数"的迷你折线图，体现增长趋势
- 同一公司行内，节点之间加淡色连线，显示演进关系

### 4. 视觉节奏感
- 2018-2022 的区域略暗/略窄，2023-2025 的区域更亮/更宽
- 体现"加速感"——越靠近现在越密集

### 5. 搜索+筛选联动
搜索和筛选可以同时生效

## 技术约束
- 仍然是单文件 HTML（index.html）+ 外部 JSON 数据（data/）
- 不引入任何 JS 框架
- 保持暗色主题
- 所有改动在 index.html 中完成
